## Homelab Docker stack

This repo is designed to be cloned onto a Linux server and bootstrapped via a modern interactive terminal wizard.

### Quick start (recommended)

Prereqs:
- Linux host with Docker + Docker Compose v2 (`docker compose`)
- `python3`
- Outbound internet (to fetch `gum` + `cloudflared` if missing)

Run the wizard:

```bash
git clone https://github.com/<you>/<repo>.git
cd <repo>
./scripts/bootstrap.sh
```

The wizard will:
- Create `.env` (from prompts) for `PUID`, `PGID`, `TZ`
- Rewrite compose files to match **your** host paths and GPU choice:
  - `[docker-compose.media.yml](docker-compose.media.yml)` (media mounts, Jellyfin URL, GPU toggles)
  - `[docker-compose.llm.yml](docker-compose.llm.yml)` (GPU toggles)
- Set up Cloudflare Tunnel via `cloudflared` CLI:
  - `cloudflared tunnel login`
  - `cloudflared tunnel create`
  - Generate `data/cloudflared/config.yml` from `data/cloudflared/config.yml.example`
  - Create DNS routes for the chosen hostnames
- Optionally create the external docker network `homelab_net`
- Validate YAML via `docker compose ... config`

### Run services

```bash
docker compose -f docker-compose.network.yml up -d
docker compose -f docker-compose.media.yml up -d
docker compose -f docker-compose.llm.yml up -d
```

### Secrets and what is (not) in git

Most `./data/*` is runtime-generated by containers and is intentionally **not** committed.

Cloudflare Tunnel secrets are **local-only** and must never be committed:
- `data/cloudflared/config.yml`
- `data/cloudflared/credentials/*.json`
- `data/cloudflared/*.pem`

This repo includes safe templates/docs under `data/cloudflared/` (for example `config.yml.example`) but keeps real secrets ignored.

### What gets rewritten

The wizard rewrites a few hard-coded defaults so a new user can set up quickly:
- Media mount prefixes `/mnt/media-hdd` and `/mnt/media-nvme`
- Plex service path `/srv/plex`
- `JELLYFIN_PublishedServerUrl`
- GPU settings (`gpus: all`, `NVIDIA_VISIBLE_DEVICES=...`) when GPU is disabled

If you prefer manual setup, copy `.env.example` to `.env` and edit the compose files directly.
